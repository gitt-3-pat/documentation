¿Qué es React?

¿Qué es React?

  • React es una libreria de JavaScript que se centra en la sintaxis declarativa y la virtualización del DOM
  • Permite a los desarrolladores escribir un JavaScript altamente eficiente con el fin de renderizar una interfaz de usuario en un navegador web
  • Mientras que el JavaScript tradicional vuelve a renderizar todo el DOM durante un cambio de estado, React sólo renderiza las partes del DOM que han cambiado

¿Qué es React?

  • React no es un framework
  • Es una librería JavaScript centrada sólo en la composición de la interfaz de usuario
  • La usarás en conjunto con otras librerías JavaScript como Node.js
  • Esto te permite construir las otras partes de tu aplicación como quieras, sin tener que comprometerte con un framework completo

Declarativo y virtualizado

Declarativo y virtualizado

  • Son realmente estas dos palabras clave las que diferencian a React del desarrollo tradicional de interfaces de usuario en JavaScript
  • React no dibuja iterativamente los componentes de la UI en la pantalla
  • El desarrollador define el aspecto final de la UI, y React se asegura de que el DOM se renderice en consecuencia
  • Esta es una diferencia muy significativa con respecto a la representación típica de HTML en JavaScript, que normalmente se centra en la inyección de HTML utilizando la directiva innerHTML

Declarativo y virtualizado

  • React también virtualiza todo el DOM y utiliza JavaScript para renderizar el HTML real en forma de React Components
  • React genera elementos HTML usando el método React.createElement()
  • Cada elemento de la interfaz de usuario renderizado en el navegador será creado por una llamada de JavaScript a este método

Declarativo y virtualizado

¿Por qué hacer eso cuando es lo suficientemente simple como para escribir HTML con JavaScript en línea?
React virtualiza todo el DOM - esto significa que mantiene un modelo dinámico de la interfaz de usuario en la memoria, y puede manipularlo sobre la marcha

Elements

  • React contiene un método para dibujar elementos HTML dentro del DOM
  • Es posible que no necesites invocarlo
  • TypeScript te da la posibilidad de escribir tus elementos en una sintaxis pseudo-HTML

Renderizado React

  • React ensamblará estas múltiples llamadas a React.createElement() en un árbol de componentes
  • En cada cambio de estado, React realiza una "tree reconciliation" donde examina el estado de cada componente y ve si algo ha cambiado
  • React actualiza esa parte del DOM virtual y vuelve a renderizar el elemento

Components

  • Los componentes son elementos atómicos de la interfaz de usuario que combinan el diseño con un estado y un conjunto de propiedades que React mantendrá
  • Estos componentes se crearán como clases o funciones de JavaScript, y React los utilizará para representar los elementos de la interfaz de usuario
  • Estos componentes son autónomos, y mantienen su propio estado y propiedades privadas
  • Pueden organizarse en jerarquías padre/hijo

Classes

  • Utilizado para componentes más complejos
  • Desarrolladores cómodos con la POO que con la programación funcional pueden preferir crear clases
  • Es una disciplina cada vez más importante a medida que avanzamos hacia arquitecturas de aplicaciones distribuidas y concurrentes

Properties

  • Los componentes contienen un objeto de propiedades con el que se puede interactuar y modificar
  • Estas propiedades se pasan por defecto como parámetros de función en un componente de función
  • Las propiedades son inmutables, pero pueden establecerse con valores iniciales cuando se renderiza un componente
